## 二、未匯入資料的具體影響

### A. 「致命」影響（不修補不能上線）

**8. 三公約對照組為 0**
- `_STEELMAN_AND_CONTROL_GROUPS.md` SOP-2 明文：「兩公約偏差須 > CRC/CEDAW/CRPD 中位數才能下『特別工具化』結論」
- 目前 D 區 5 張 PI 的所有觀察都**無法成立「兩公約特別被工具化」結論**
- 風險：任何 D 區結論被學界一秒戳破

### B. 「高」影響（須在 v1.0 前完成）

**1. CO 6% 覆蓋率**
- PI-02 雖有 15 條 critical CO，但其餘 235 點完全未進 SQLite
- 無法做「條文 × CO × 政府執行率」交叉表
- 影子報告 v0.2 之 PI-01 摘要無底氣

**2. 影子報告全文 0 份**
- 「誰倡議了什麼」依然是模糊回答
- trace 系統雖建構好，但事件節點密度仍稀薄

**3. NAP I 154 項 0 匯入**
- 「政府執行率」是 PI-05 / PI-06 之核心
- 目前只有 metadata，無法逐項追蹤

**5. 立法院公報 < 0.1%**
- PI-13 試辦 8 件案例尚不足以做 chi-square / Gini
- 須對接立法院 API（須 token）+ PDF OCR

**6. 司法判決援引 0 筆**
- 黃昭元觀察「大法官援引比例約 1%」之數字無法獨立驗證
- 須對接司法院法學資料檢索系統

### C. 「中」影響（可分批補）

**7. 委員 30% 覆蓋**
- 第 4 屆 12 位有完整資料
- 1-3 屆 27 位次只有姓名，無學術立場 / 過去審查紀錄
- 可分批用 Felo / WebFetch 補

**9. 預算 4% 覆蓋**
- 須至主計總處逐年下載 PDF 預算書
- 半自動化可做（pdfplumber + regex）

**10. 媒體 archive 0 件**
- 須對 web.archive.org 主動觸發 save
- 可寫腳本批次跑

## 六、誠實結論

**目前狀態最適合定位為**：
- ✓ 「**研究計畫骨架** + 部分實證」
- ✓ 「**對學界公開之 BETA 版本**」
- ✓ 「待外部審查之早期文獻」
- ✗ 「**完整影子報告**」（尚不足）
- ✗ 「**對社會大眾之主張平台**」（尚不足）

平台目前**最該做的事**並非加更多功能，而是：
1. 補核心資料缺口（CO 全文、NAP 行動、對照組）
2. 完成法律顧問複核
3. 啟動外部審查邀請
4. 建立用戶反饋機制（Tally）

詳見 `docs/ROADMAP_TO_V1.md`（下個檔案）。
