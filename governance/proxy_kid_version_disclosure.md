# 兒少版內容之 Proxy 撰寫透明聲明

> **由獨立 QA 第二輪審查觸發(2026-04-26)**:CRC2-CO 強調「**by children, not for children**」,但本平台 14 議題卡片之「兒少版」段落目前皆為**成人代為撰寫(proxy 撰寫)**,未經兒少自身書寫或審稿。本檔建立透明聲明 + 後續改善路徑。

## 一、現況(2026-04-26 Wave 45 後)

| 議題 | 兒少版字數 | 撰寫者 | 兒少審稿? |
|---:|---:|---|---|
| PI-01 ~ PI-14 | **多數 ≤200 字**(Wave 45 已將 8 篇超長段落瘦身) | 平台編輯(成人) | ❌ **全部仍未經兒少審稿** |

**Wave 45 進展(2026-04-26 後續)**:
- 14 個 PI frontmatter 一律加 `kid_version_status: 成人代擬,待兒少審稿小組(8 人)重寫`(metadata 透明化)
- 8 篇超 200 字兒少版瘦身至 ≤200 字:PI-04、PI-05、PI-06、PI-08、PI-09、PI-10、PI-13、PI-14
- self_qa.sh 字數計算修正(原 wc -m 數 bytes,改 Python 數 chars)
- 此瘦身**僅作為兒少審稿小組之輸入草稿**,實質改寫仍待 8 人小組

## 二、誠實揭露(平台應對外公開)

於兒少版頁面顯著位置加註:

```
這頁是大人為孩子寫的版本(proxy 撰寫)。
我們在做的事是讓你能看懂、能參與,但目前還沒有真正讓孩子來改、來寫。
如果你想告訴我們「這頁哪裡寫得不夠好」「應該怎麼寫」,
請點 [我想說一件事] 表單,或在你 13 歲以上時加入「兒少審稿小組」。
```

(實作:在 `_jekyll_output/templates/_layouts/kid.html` 之兒少版頁面 header 強制顯示此聲明)

## 三、改善路徑(分階段)

### 階段 A — 透明聲明(立即,Wave 31)
- 本檔建立 ✅
- 兒少版頁面 header 加註 proxy 聲明(待 Wave 32 加入 kid.html 模板)

### 階段 B — 兒少審稿機制(2026-Q3 啟動)
- 結合 [`forms/child_15_18.md`](../forms/child_15_18.md) 之「兒少顧問小組」邀請欄位
- 邀請 8 名(4 國中 + 4 高中)兒少審稿員
- 14 議題卡片每月分 1-2 個交給兒少審稿員,提供書面回饋
- 平台公布每次回饋與後續修改紀錄(對應 Lundy Model 之 "Influence" 閉環)

### 階段 C — 兒少撰寫共製(2027-Q1 啟動)
- 議題卡片之「兒少版」改為**兒少自己寫初稿、成人協助潤飾**
- 對應 CRC GC12 §132-134(兒少參與決策)、CRC2-CO §52(LGBTI 兒少表意)
- 在影子報告之撰寫方法論段落明確說明此進展

## 四、與既有政策之對接

- [`governance/content_sop.md`](content_sop.md) 之「兒少版 SOP」加註本透明聲明要求
- [`forms/ethics_checklist.md`](../forms/ethics_checklist.md) 第 26-28 項(影響力閉環)涵蓋此議題
- [`PI-10 兒少表意與校園民主`](../data/policy_issues/PI-10_兒少表意與校園民主.md) 加 R-PI10-X「平台自身先做榜樣」之政策建議

## 五、為何重要

CRC GC12 之核心精神是兒少**作為權利主體之表意能力**,而非被代為發聲。若平台本身未做到,在影子報告中對政府要求 CRC-12 落實時,將失去道德高度。**這是平台必須誠實面對的內部矛盾,並以分階段改善公開承擔之**。
