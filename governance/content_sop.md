# 內容生產 SOP

> **適用範圍**:議題卡片、影子報告章節、媒體稿、社群貼文、兒少版內容

---

## 一、議題卡片 7 步流程

1. **選題**(主編或外部建議)→ 加入 backlog
2. **研究助理填寫 12 欄結構化資料**(2-3 工作日)
   - 對應 PI-XX 議題卡片 YAML frontmatter + 全部章節
3. **主編審核並補強政策建議**(1 工作日)
4. **外部專家審閱**(可選,3-5 工作日)
5. **兒少友善版改寫**(reading age 對齊國小六年級)
6. **DPO 隱私與倫理檢核**(若涉案例,必做;否則抽查)
7. **發布**:Super.so 公開頁 + GitHub Pages 主站 + 同步 SQLite

## 二、議題卡片更新週期

| 欄位類型 | 頻率 |
|---|---|
| 統計指標 | 季度 |
| 政策追蹤(政府回應、進度狀態)| 月度 |
| CRC 條文 | 無變動可常駐(GC 發布時更新)|
| 案例故事 | 年度盤點 + 個案隨時新增 |

## 三、影子報告生產 SOP(6-8 個月)

1. **議題盤點**:從平台拉出所有 PI-XX 議題卡片,標記 cluster
2. **召開民間共識會議**:視訊 + 實體,3 場
3. **議題執筆分工**:每位執筆人 2-3 個議題
4. **兒少參與工作坊**:2 場,8-14 歲一場、15-18 歲一場
5. **草稿匯流**:`python shadow_report/compile.py` 自動產出 + 人工潤飾
6. **同儕審查**:邀請台灣兒童權利公約聯盟等聯盟成員交叉審閱
7. **翻譯與校對**:中譯英(專業審查委員為英文母語讀者)
8. **提交**:衛福部社家署 + 國家人權委員會 + 國際審查委員

## 四、媒體稿生產 SOP

從平台議題卡片或影子報告,擷取核心數據與案例,以 800-1200 字撰寫:
- **第一段**:三個核心衝突數字
- **中段**:制度缺口與政策反證
- **結尾**:2-3 項可立即啟動的政策建議

由主編草擬 → 法遵檢查(無個資外露)→ 對外發布。

## 五、兒少版 SOP

由議題卡片的「兒少版(80 字)」段落擴寫為頁面內容。**禁止**:
- 直接翻譯成人版
- 使用法律或政策術語
- 描述創傷細節
- 提及具體個案足以辨識者

**必須**:
- 提供求助資源(113、1925、1980、iWIN)
- 提供「我想說」表單入口
- 提供「快速離開」按鈕
- 提供白話 + 視覺輔助(插畫、語音版)

## 六、品質檢核清單(發布前)

- ☐ YAML frontmatter 完整,所有欄位皆填或標 `待補`
- ☐ 所有引用來源可回溯(URL 或文件名 + 頁碼)
- ☐ 數據引用註明日期(避免過期數據誤導)
- ☐ 政策建議標註對象、期程、可檢核標準
- ☐ 兒少版 reading age 適齡(可請第三方測試)
- ☐ 隱私政策連結存在
- ☐ 兒少報導倫理紅線(`child_safeguarding.md`)逐項通過
- ☐ 無 broken link
- ☐ Schema.org JSON-LD 已嵌入
