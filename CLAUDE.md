# 兩公約施行總檢討平台 — 專案指令

> 專案啟動日：2026-04-29
> 方法論來源：兒少權監督平台 Wave 1-92(`docs/PLATFORM_PLAYBOOK.md`)
> 隸屬組織：國教行動聯盟（AABE）青年部 / 政策倡議室
> 部署目標：`policy.aabe.org.tw/two-covenants/` 子目錄 或 獨立子網域

## 一、專案目的（一句話）

蒐集、比對、公開「兩公約（ICCPR + ICESCR）在台灣施行 16 年（2009-2026）」的全部官方文件、結論性意見、國家人權行動計畫進度、國際比較、以及**兩公約被特定政黨與運動團體工具化的客觀證據**，讓使用者用這個平台**快速應對與倡議**：兩公約原本應保障的普世人權，不應變相成為單一議程（同志運動、政黨工具）的擋箭牌。

## 二、核心立場（紅線，不可偏離）

1. **支持兩公約原始精神**：保障普世人權（生命、人身自由、思想信仰、平等、健康、教育、勞動、文化等）。
2. **反對工具化**：反對將兩公約曲解為**單一議題（特別是 LGBTQ+ 運動 / 同性婚姻 / 性別認同議程）的法律憑藉**，因為兩公約原文與一般性意見並未要求締約國採取此立場。
3. **反對政黨化**：反對**特定政黨（不點名，由證據說話）**將兩公約結論性意見選擇性援引，作為自身政策合法化工具，卻迴避其他更重大的人權問題（例如：兒少自殺率、原住民權益、勞動條件、宗教自由、言論自由空間萎縮）。
4. **客觀證據優先**：所有指控必須附條文號 + 官方文件出處 + 時間戳。**不得使用情緒性語言**。
5. **尊重宗教與良心自由**：ICCPR 第 18 條（思想、良心和宗教自由）必須與第 26 條（不歧視）並重，不得以後者吞沒前者。
6. **不攻擊特定個人 / 性傾向當事人**：對事不對人，批判的是**運動團體與政府機關的政策操作**，不是個別當事人的人格。

## 三、雙軸審查框架（套用本平台）

### 類別一：過去要求沒兌現
- ICCPR / ICESCR 兩公約**歷次國際審查**結論性意見（CO）
  - 第一次審查 2013 年（Manfred Nowak 等 10 位委員）
  - 第二次審查 2017 年
  - 第三次審查 2022 年
  - 第四次審查 2025 年（最新）
- **國家人權行動計畫（NAP）**：第一期 2022-2024、第二期 2025-2028 進度
- **NHRC 監察院國家人權委員會**獨立監督報告

### 類別二：關注議題沒提（被工具化的證據）
- 結論性意見**被選擇性援引**的條目（哪些被高調推、哪些被埋）
- 兩公約**原始一般性意見（General Comments）**對比台灣實踐落差
- 重大實際**未進步**的證據：
  - 兒少自殺率上升（衛福部數據）
  - 國民健康下降（菸毒、毒駕）
  - 言論自由空間（NCC 處分爭議、新聞自由指數）
  - 宗教自由（特定團體監控、稅務查核選擇性）
  - 勞動權（過勞死案件、移工權益）
  - 司法獨立（法官評鑑、政治介入）
  - 原住民傳統領域權
- **政黨化與工具化證據**：
  - 哪些立委 / 政黨 / 運動團體在哪場合引用兩公約
  - 對應的法案 / 政策推進實際內容
  - 與兩公約原文的落差比對

## 四、5 種輸出版本

| 版本 | 受眾 | 用途 |
|---|---|---|
| 專業版 | 審查委員、政府、學者 | 含條文號 + 一般性意見 + 統計 |
| 媒體版 | 記者、公眾 | 800-1200 字，先講結論 |
| 立法院版 | 立委辦公室 | 質詢題草稿 + 修法建議 |
| 倡議版 | 教會、家長團體、其他 NGO | 行動指引 + 速查卡 |
| 社群版 | IG / FB / Threads | 140 字 + 圖卡腳本 |

> 本專案**不設兒少版**（議題對 8-18 歲過於抽象），改設「倡議版」給合作夥伴用。

## 八、絕對紅線（不可違反）

1. ❌ **不點名特定當事個人**（除非已被法院判決公開）
2. ❌ **不使用情緒性 / 攻擊性語言**（例：邪惡、可恥、變態）
3. ❌ **不偽造引用**（每段論述必有 URL 或出版資訊）
4. ❌ **不公開未授權的個人資料**
5. ❌ **不在沒讀 governance/transcript_citation_sop.md 前處理任何訪談**
6. ❌ **不使用 emoji 在公開 HTML**（用 inline SVG）
7. ❌ **不 force push、不 amend 已 push 的 commit**
8. ❌ **不安裝 brew/pip 系統套件**（用 .venv/bin/pip）
9. ❌ **不主動加 Notion / Super.so**（堅持 Markdown SSOT）
