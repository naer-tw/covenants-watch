議題脈絡追溯

本平台脈絡追溯設計紅線（5 條）：
1. 稱「脈絡追溯」非「責任追溯」（記論述鏈，不判責任）
2. 每個歸因節點必附反證 / 反事實
3. 因果鏈不超過 3 層（避免過度推論）
4. 正反結果並陳
5. 論述（statement）與責任歸屬（accountability）嚴格分離

四種查詢模式

用 scripts/two_cov_trace.py 命令列查詢：

# 1) 議題向前追溯
python3 scripts/two_cov_trace.py --issue 廢死

# 2) 行動者全紀錄（誰倡議了什麼？政府回應什麼？）
python3 scripts/two_cov_trace.py --actor "廢死聯盟"
python3 scripts/two_cov_trace.py --actor "Manfred Nowak"

# 3) 結果反推：給一個 outcome event_id，列出可能影響它的事件鏈（含反證）
python3 scripts/two_cov_trace.py --backtrack EV-2024-002

# 4) 全文檢索
python3 scripts/two_cov_trace.py --search "OP2 批准"

使用情境範例

情境 A：「我看到一個議題」

例：你想了解「兒少自殺率上升」的脈絡。

  1. 進入 youth_suicide.html（待後續填充）查看完整時序鏈
  2. 看哪個 NGO 在哪屆審查倡議何政策、產出何 CO
  3. 看政府後續修法、預算配置、實際指標變化
  4. 每條因果鏈都附「反證」+「並列原因」，避免單向歸因

情境 B：「我看到一個負面結果」

例：你看到「青少年自殺率 2023 年高於 OECD 平均」，想反推。

  1. 找對應 outcome event_id
  2. 跑 --backtrack EV-XXXX-XXX
  3. 系統列出**所有可能的上層論述鏈**，每條附反證與並列原因
  4. 鏈長 > 3 自動加警告（避免過度推論）
  5. 讀者依「反證 / 並列原因 / 多元因果」自行評估

情境 C：「我想了解一個倡議者」

例：你想知道「Manfred Nowak 4 屆都關注什麼？」

  1. 跑 --actor "Nowak"
  2. 系統列出該行動者全部論述記錄（按時序）
  3. 含立場光譜、活躍期、所有事件

資料結構（4 表 + FTS5）

  • actor：行動者（NGO / 委員 / 政府 / 立委 / 媒體 / 學者 / 法院 / 民眾）+ 立場光譜欄位
  • event：論述 / 行動 / 結果（含 issue_tags、related_pi、related_co、source_url、source_archive、is_positive_outcome）
  • causal_link：因果連結 + 反證 + 並列原因 + 鏈深度
  • outcome_indicator：結果指標（前後值 + 混淆變項清單）
  • SQLite views：v_issue_timeline、v_actor_record（簡化查詢）
  • FTS5：fts_event 全文檢索

編碼品質

  • 每筆 causal_link 須由 ≥ 2 名研究員獨立編碼後仲裁
  • 編碼者間信度（kappa coefficient）≥ 0.7 才能對外發布
  • 所有反證 / 並列原因公開供第三方挑戰
