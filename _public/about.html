一、平台宗旨

本平台聚焦兩公約（公民與政治權利國際公約 ICCPR、經濟社會文化權利國際公約 ICESCR）於 2009 年通過《兩公約施行法》以來 16 年（2009-2026）之施行歷程。

使用者透過本平台可：

  • 檢視四次國際審查（2013/2017/2022/2025）約 250+ 條結論性意見之執行落差
  • 追蹤兩期國家人權行動計畫（NAP）之承諾與實際進度
  • 對照國際標準與台灣實踐之差距（含一般性意見、各國判例）
  • 用客觀資料檢驗條文援引、立法院公報、白皮書、教科書等公領域之分布特徵

二、平台立場（紅線）

1. 支持 兩公約原始普世人權精神（生命、自由、平等、健康、教育、勞動、宗教等）

2. 反對 將兩公約曲解為單一議程（特別是 LGBTQ+ 運動 / 同婚 / 性別認同）的法律憑藉

3. 反對 特定政黨將結論性意見選擇性援引，作為自身政策合法化工具

4. 客觀證據優先：所有指控必須附條文號 + 官方文件出處 + 時間戳

5. 尊重宗教與良心自由：第 18 條須與第 26 條並重，不得被吞沒

6. 對事不對人：批判政策操作，不點名個別當事人人格

三、研究方法（雙軸 + 對照組）

3.1 雙軸審查框架

  • 類別一：過去要求沒兌現 — ICCPR/ICESCR 4 屆審查 CO + 兩期 NAP + NHRC 報告
  • 類別二：援引行為與政策落差客觀檢驗 — 條文援引頻率、政黨選擇性、政府預算配置

3.2 三公約對照組（防止單獨歸因）

任何「兩公約被特別工具化」之結論，必須同步對 CRC（兒童權利公約）、CEDAW（消除對婦女歧視公約）、CRPD（身心障礙者權利公約）跑相同手法。
只有當兩公約之偏差 > 三個對照組偏差中位數時，才能下「兩公約被特別工具化」結論。

3.3 Steelman 規範

每張議題卡皆含「§Steelman 段落」，完整呈現本平台反對立場之最強論述，引用其最具代表性之學術 / 倡議文獻；本平台之反論置於其後。

四、品質控管

  • 預先註冊：研究設計於資料蒐集前公開，事後不得修改度量指標（避免 p-hacking）
  • 原始資料公開：所有抓取結果以 SQLite + CSV 雙格式公開
  • 第三方可重現：所有腳本以 BSD-3 開源
  • 外部審查：發布實證結論前邀請 ≥3 位審查者（含 1 位反對立場學者）
  • 異議申訴：被認為「不對稱」之團體可在報告發布後 30 日內提出方法論挑戰，若成立則修訂結論

五、發布原則（避免名譽風險）

  • 不點名特定當事個人（除已被法院判決公開）
  • 政黨資訊以中性編碼處理，分析時以政黨代碼（P1/P2/...）呈現
  • 所有具體案號 / 個案須經法律顧問複核 + 14 日異議期始得公開
  • NGO 立場分類由 ≥ 2 名研究員獨立編碼，受訪 NGO 可請求修正分類
  • 所有外部引用之網址必有 archive.org 典藏

六、平台技術架構

  • 後台編輯：Markdown + git（版本控制 + 不鎖定）
  • 主資料庫：SQLite（FTS5 全文索引）
  • 部署：GitHub Pages（靜態 + CDN）
  • 月運作成本：USD 0

七、組織歸屬
